# 스레드(Thread) 기초 정리

## 1) 스레드가 뭐야?
`스레드`는 한 **프로세스** 안에서 실제로 실행되는 가장 작은 작업 단위야.

쉽게 말해:
- **프로세스** = 실행 환경(공유 자원 묶음)
- **스레드** = 그 환경을 사용하는 작업자(작은 실행 흐름)

## 2) 스레드와 프로세스의 차이

### 프로세스
- 실행 중인 프로그램 하나
- 고유한 메모리 공간(코드, 데이터, 힙, 열린 파일, 환경 변수 등)을 가짐
- 서로 다른 프로세스는 기본적으로 자원을 거의 안 공유함

### 스레드
- 프로세스 내부에서 동작
- 같은 프로세스 안의 스레드끼리는 **메모리·파일 같은 자원 공유**
- 각 스레드는 자신만의 실행 흐름(코드 실행 위치), 레지스터, 스택만 따로 가짐

## 3) 비유로 이해하기
집(프로세스)에 `주방`, `침실`, `거실`이 있다고 생각해 봐.

- 집 전체 = 프로세스 (공유되는 집)
- 방 하나씩 = 스레드 (각자 다른 일을 하고 있음)
- 하지만 모두 집 안의 부엌, 전기, 화장실(공유 자원)을 같이 씀

## 4) 한 번에 왜 여러 스레드가 필요해?

스레드는 **한 번에 여러 일을 할 수 있게** 만들어져.

예시:
- 웹 브라우저에서:
  - 하나의 스레드는 화면을 그리기
  - 다른 스레드는 네트워크 데이터 수신
  - 또 다른 스레드는 사용자 입력 처리

이렇게 나누면 반응성이 좋아지고 효율이 높아져.

## 5) 스레드가 왜 어려워 보이나?

쉬운 이유와 어려운 이유가 같이 있어.

- 쉬운 이유: CPU에서 여러 작업을 동시에 처리하는 느낌을 주기 좋음
- 어려운 이유: 공유 자원을 동시에 건드릴 때 **문제(동시성 문제)**가 생길 수 있음

주의할 점 예시:
- 두 스레드가 같은 변수 `counter`를 동시에 바꾸면 값이 꼬일 수 있음
- 이를 해결하려고 락(lock), 세마포어, 동기화 기법을 사용

## 6) 간단 정리
- 스레드는 프로세스 안의 실행 단위
- 스레드 간에는 자원을 공유함
- 여러 스레드 = 병렬/동시 작업 가능
- 장점: 반응성 향상, 자원 효율성
- 단점: 동시성 버그(경쟁 조건, 데드락) 가능성

## 7) 한 줄 요약
**스레드는 "한 프로세스 안에서 일하는 여러 작업자"이고, 프로세스는 그들이 함께 쓰는 작업장이다.**
