## 4월 18일(토): 핵심 개념 + Echo 서버 완성
### 목표
- 네트워크/소켓/HTTP의 큰 그림을 먼저 잡고, echo 서버를 구현 및 검증합니다.

### 학습 순서
1. 네트워크 개념 & TCP/IP 계층
2. 소켓 인터페이스 전체 흐름(socket, bind, listen, accept, connect, close)
3. 클라이언트-서버 모델 + 파일 디스크립터
4. Datagram vs Stream Socket 비교

### 구현/테스트
1. echo 서버: `open_clientfd()` -> `open_listenfd()` -> 클라이언트 `main()` -> 서버 `main()` -> `echo()`
2. telnet 테스트: 입력 내용이 동일하게 응답되는지 확인

### 산출물
- echo 서버 구현 코드
- 테스트 로그(정상/비정상 1개 이상)

## 4월 20일(월): Tiny 서버 집중 구현 + 1차 통합
### 목표
- tiny 서버 주요 함수 구현을 끝내고 정적/동적 컨텐츠 동작을 확인합니다.

### 학습 순서
1. 웹서버 기본 동작
2. 웹컨텐츠(MIME, 정적/동적, CGI)
3. HTTP(요청/응답, 헤더, 메소드, 상태코드, HEAD)

### 구현 순서
1. `main()`
2. `doit()`
3. `read_requesthdrs()`
4. `parse_uri()`
5. `serve_static()`
6. `get_filetype()`
7. `serve_dynamic()`
8. `clienterror()`
9. cgi `adder` 구현

### 테스트
1. telnet으로 tiny HTTP 버전 확인
2. 브라우저에서 `home.html` 요청/응답 확인
3. 브라우저에서 `cgi-bin/adder?123&456` 요청/응답 확인

### 산출물
- tiny 서버 구현 코드
- 정적/동적 컨텐츠 테스트 증빙

## 4월 21일(화): Proxy 구현 + 캐시/동시성 + 퀴즈 대비
### 목표
- proxy 서버를 순차 -> 동시병렬 -> 캐싱 순으로 완성하고, 핵심 키워드/코드 퀴즈 대비를 마무리합니다.

### 구현 순서
1. proxy 서버 순차 처리 구현
2. proxy 서버 동시병렬 처리 구현
3. 캐싱 웹 오브젝트 구현

### 테스트 순서
1. PC 네트워크 프록시 설정
2. 로컬 -> 프록시 -> tiny `home.html` 성공 확인
3. 원격 프록시 중지 후 실패 케이스 확인
4. basic test(순차 처리)
5. concurrency test(동시병렬)
6. cache test(캐시 적중/미적중)

### 퀴즈 대비 (화요일 오전)
